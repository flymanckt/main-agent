# Agent 协同规则

## 默认入口
所有问题默认先由 `biz-architect` 接收。

## 分流规则
### 继续由 biz-architect 直接处理
适用于：
- 业务流程梳理
- 痛点诊断
- 管理问题分析
- 流程优化建议

### 引入 solution-architect
适用于：
- 系统边界划分
- 模块职责设计
- 数据流与接口流设计
- 多系统协同方案

### 引入 delivery-consultant
适用于：
- 项目分期
- 实施优先级
- 风险控制
- 上线切换
- 主数据与权限治理

## 统一输出原则
无论哪个 agent 参与，最终都必须回到统一格式：
1. 结论
2. 现状问题
3. 根因判断
4. 方案建议
5. 优先级/实施建议
