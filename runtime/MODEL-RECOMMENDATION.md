# Finance 模型与数据源建议

## 结论
当前 finance 准确率的主要瓶颈是数据层，不是模型层。

推荐方案：
1. 日常盘前 / 午间 / 收盘摘要：`minimax/MiniMax-M2.7`
2. 深度复盘 / 周复盘：`openai-codex/gpt-5.2`
3. 行情源优先做双源：`eastmoney + akshare`

## 原因
- 当前系统主要是规则脚本，不是重度 LLM 驱动。
- 免费东财接口适合辅助监控，不适合单源决策。
- MiniMax-M2.7 成本低，中文总结性价比高，适合做解释层。
- gpt-5.2 更适合复杂复盘，不适合每次普通巡检都使用。

## 推荐架构
1. 数据层：东财 + AkShare + 后续公告/新闻源
2. 规则层：止损、涨跌幅、异常值校验、观察池评分
3. 模型层：仅负责解释、总结、归因，不直接裸做行情判断

## 配置建议
- 默认模型：`minimax/MiniMax-M2.7`
- 深度分析模型：`openai-codex/gpt-5.2`
- 默认不开启 LLM 总结，先把结构化数据质量打稳
- 环境允许后再开启 `FINANCE_ENABLE_AKSHARE=1`
