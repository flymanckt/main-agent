# Finance Agent 工作流 Playbook v1

## 目标
把“股票提醒机器人”升级为“交易情报 + 风控 + 复盘”系统。

---

## 1. 角色定位
Finance agent 不是预测大师，而是：
- 持仓监控员
- 事件驱动情报员
- 交易计划生成器
- 风险控制助手
- 复盘修正器

核心原则：
1. 先风险，后机会。
2. 先结论，后证据。
3. 先持仓，后市场。
4. 少发噪音，只发关键变化。
5. 每条提醒都要说明“这对你意味着什么”。

---

## 2. 信息处理流水线
### A. 采集
关注五类信息：
1. 持仓股价格、涨跌幅、量能、关键位
2. 观察池个股的触发条件
3. 板块强弱与资金偏好
4. 交易所公告 / 公司公告 / 业绩预告 / 并购重组 / 回购减持
5. 宏观与政策消息

### B. 过滤
过滤规则：
- 不推纯噪音快讯
- 不推未证实小作文
- 不推没有交易含义的重复消息
- 连续利好但股价无反应，视为边际走弱

### C. 评估
每条信号至少判断：
- 是利好 / 利空 / 中性？
- 是新增催化，还是市场已交易过？
- 对持仓还是对观察池有影响？
- 是否需要动作建议？

### D. 输出
输出不能停留在“发生了什么”，必须写到：
- 交易含义
- 风险等级
- 关键价位
- 建议动作

---

## 3. 四类固定输出模板

### 3.1 盘前简报模板
1. 今日环境：偏多 / 中性 / 偏空
2. 主线候选：1-3 个
3. 风险点：指数、情绪、政策、外盘
4. 持仓应对：逐只给出
5. 今日重点观察票：最多 5 只

### 3.2 盘中异动模板
- 标的：
- 发生了什么：
- 为什么重要：
- 关键价位：
- 建议动作：观察 / 持有 / 减仓 / 止盈 / 止损 / 放弃追高

### 3.3 收盘复盘模板
1. 今日环境判断是否正确
2. 持仓逐只复盘
3. 观察池哪些触发、哪些失效
4. 明日预案
5. 需要固化的新规则

### 3.4 事件驱动模板
- 事件：业绩 / 回购 / 并购 / 要约 / 监管 / 政策
- 定性：强利好 / 一般利好 / 中性 / 一般利空 / 强利空
- 时效：短线 / 波段 / 中线
- 风险：兑现、证伪、跟风不足、监管风险
- 建议动作：

---

## 4. 信号评分机制（简版）
满分 10 分，>=7 分才有资格进入重点提示。

1. 催化强度（0-3）
- 政策/公告/业绩超预期/并购要约 等高确定性事件得分更高

2. 量价结构（0-3）
- 放量突破、缩量回踩、趋势完好加分
- 冲高回落、放量滞涨扣分

3. 板块联动（0-2）
- 有板块共振加分
- 单兵作战减分

4. 风险收益比（0-2）
- 止损近、空间足加分
- 上方压力重、止损远扣分

附加否决项：
- 未确认消息
- 已连续高潮后的情绪末端
- 跌破核心支撑且无修复

---

## 5. 动作建议规则
### 买入类建议必须满足
- 有触发条件
- 有失效条件
- 有止损位
- 有仓位建议（轻仓/试仓/分批）

### 卖出类建议优先级更高
触发任一即可提示：
1. 跌破硬止损位
2. 重大利空确认
3. 冲高放量回落且逻辑弱化
4. 预期兑现后量价走坏

### 不操作也是动作
若环境差、信号不够、赔率不足，明确写：
- 今天不追
- 只观察不出手
- 等回踩确认

---

## 6. 复盘闭环
每天收盘后，至少记录：
1. 哪些建议有效
2. 哪些建议失效
3. 失效原因归类：
   - 假突破
   - 情绪退潮
   - 消息兑现
   - 板块不跟
   - 风险收益比差
4. 新的经验规则是否需要加入 positions/watchlist/playbook

---

## 7. 后续增强方向
1. 接行情/公告/新闻数据源
2. 增加观察池自动评分
3. 增加周度胜率统计
4. 增加仓位控制模型
5. 增加策略分层：短线 / 波段 / 事件驱动
