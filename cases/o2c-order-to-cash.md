# O2C（销售到回款）案例包

## 场景
企业希望从客户管理、商机、报价、合同、订单、发货、开票、应收、回款形成闭环，提高收入转化与资金回笼效率。

## 常见痛点
1. CRM 只管销售线索，不管订单与回款
2. 合同、订单、发货、开票数据断裂
3. 销售、财务、交付口径不一致
4. 应收账款催收靠人工盯
5. 回款分析滞后，难以识别风险客户

## 根因判断
- CRM 与 ERP 缺少主线打通
- 客户、合同、订单主数据不统一
- 信控与授信规则未嵌入流程
- 财务与业务对收入确认口径不一致

## 典型系统分工建议
- CRM：客户、商机、报价、销售预测
- OA：通用合同审批（如不承载核心业务控制）
- ERP：订单、发货、开票、应收、收入确认
- 资金系统：回款识别、资金归集、银企对账

## 推荐优化思路
1. 建立客户—合同—订单—应收主链路
2. 前置信控与授信规则
3. 建立回款责任机制与预警机制
4. 统一 CRM 与 ERP 的数据口径
5. 建立订单到现金的全过程可视化

## 顾问输出重点
- 当前断在商机、合同还是回款
- CRM 与 ERP 边界是否合理
- 回款控制应放在哪个环节
- 哪些指标能真正反映 O2C 质量
