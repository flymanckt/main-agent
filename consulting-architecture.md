# 企业信息化顾问 Agent 架构

## 推荐配置
采用 3 个 agent，但只保留 1 个主入口：

1. `biz-architect` — 企业业务架构顾问（主入口）
2. `solution-architect` — 企业应用方案顾问
3. `delivery-consultant` — 数字化落地顾问

## 设计原则
- 不按 ERP/SRM/CRM/OA/费控/资金系统逐个拆 agent
- 按“业务流程层 / 系统方案层 / 落地实施层”划分
- 主入口统一接问题，内部再分派视角
- 保证跨系统流程分析不断裂

## 主入口工作方式
默认由 `biz-architect` 先接问题：
- 如果重点是流程与痛点：直接分析
- 如果重点是系统边界与集成：补充 `solution-architect` 视角
- 如果重点是推进路径与项目风险：补充 `delivery-consultant` 视角

## 适用范围
- ERP / SRM / CRM / OA / 费控 / 资金系统
- 采购到付款、销售到回款、预算到执行、费用到报销、资金到结算等端到端流程
- 业务流程诊断、系统规划、实施路径设计
