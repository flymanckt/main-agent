# 成熟度评估框架

## 评估维度
### 1. 流程成熟度
- L1：靠人推动，缺少标准
- L2：有流程，但执行不稳定
- L3：流程基本标准化
- L4：流程可监控、可度量
- L5：流程持续优化并可复制

### 2. 系统成熟度
- L1：Excel / 邮件 / 人工为主
- L2：已有系统，但边界混乱
- L3：核心流程已有主系统承载
- L4：关键系统已集成
- L5：系统边界清晰，数据贯通

### 3. 数据成熟度
- L1：多口径、多版本
- L2：有主数据但不稳定
- L3：关键主数据基本统一
- L4：数据质量可控
- L5：数据可用于经营决策闭环

### 4. 管控成熟度
- L1：靠经验管理
- L2：有审批但控制点后置
- L3：关键规则已嵌入流程
- L4：预算/权限/合同/付款联动控制
- L5：形成前置预警与持续优化机制

### 5. 落地成熟度
- L1：没有明确路线图
- L2：有想法但无统一优先级
- L3：能做一期切入
- L4：有分阶段规划和治理机制
- L5：可持续复制推广

## 输出建议
每次评估后给出：
1. 当前成熟度等级
2. 主要短板
3. 一期优先动作
4. 未来提升路径
